# Deployment Guide - Biogas Telegram Bot

## 🚀 Quick Deploy to Render (15 minutes)

### Prerequisites Checklist

- [ ] GitHub account
- [ ] Telegram bot token (from @BotFather)
- [ ] Google Cloud account (for Service Account)
- [ ] Google Sheet created with correct schema
- [ ] Git installed locally

---

## Step 1: Create Telegram Bot

1. Open Telegram and search for `@BotFather`
2. Send `/newbot` command
3. Follow prompts:
   - Bot display name: `Biogas Data Bot`
   - Bot username: `biogas_data_bot` (must end in 'bot')
4. **Save the bot token** (looks like: `123456789:ABCdefGHIjklMNOpqrSTUvwxYZ`)

### Add Bot to WhatsApp Group (via Forwarding)

1. Create a Telegram group for your team
2. Add the bot to the group
3. Make the bot an admin (optional, for reading all messages)
4. Instruct staff to forward WhatsApp messages to this group every ~2 hours

---

## Step 2: Setup Google Sheets API

### 2.1 Create Google Cloud Project

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create new project: `biogas-telegram-bot`
3. Enable these APIs:
   - Google Sheets API
   - Google Drive API

### 2.2 Create Service Account

1. Navigate to **IAM & Admin** → **Service Accounts**
2. Click **Create Service Account**
   - Name: `biogas-bot-sheets`
   - Grant role: **Editor**
3. Click **Done**

### 2.3 Generate Credentials

1. Click on the service account email
2. Go to **Keys** tab
3. **Add Key** → **Create new key** → **JSON**
4. Download and save as `credentials.json`
5. **Keep this file secure!**

### 2.4 Prepare Google Sheet

1. Create a new Google Sheet
2. Create a worksheet/tab named `Complaints Register`
3. Add header row (Row 1):

```
A1: Complaint ID
B1: Date Reported
C1: Customer Name
D1: Customer ID / Account
E1: Phone Number
F1: JBL Reported By
G1: Branch / Region
H1: Complaint Category
I1: Complaint Description
J1: LOAN STATUS
K1: LOAN AT RISK
L1: Status
M1: Resolution Details
N1: Date Resolved
O1: Days Open
P1: RISK LEVEL
Q1: Internal Message ID
R1: Parsed Timestamp
```

3. **Share the sheet** with the service account email (from credentials.json)
   - Click **Share** → Add service account email → **Editor** permission
4. **Copy the Sheet ID** from URL:
   ```
   https://docs.google.com/spreadsheets/d/<SHEET_ID>/edit
   ```

## Step 3: Deploy to Render

### Option A: One-Click Deploy (Recommended)

1. **Push code to GitHub:**
   ```bash
   cd biogas_bot
   git init
   git add .
   git commit -m "Initial commit: Biogas Telegram Bot MVP"
   git remote add origin <your-repo-url>
   git push -u origin main
   ```

2. **Connect to Render:**
   - Go to [render.com](https://render.com)
   - Sign up/login with GitHub
   - Click **New +** → **Blueprint**
   - Connect your GitHub repository
   - Render will auto-detect `render.yaml`

3. **Configure Environment Variables:**
   Render will prompt for these:
   ```
   DJANGO_SECRET_KEY: (auto-generated by Render)
   DEBUG: false
   ALLOWED_HOSTS: your-app-name.onrender.com
   TELEGRAM_BOT_TOKEN: 123456789:ABCdef...
   TELEGRAM_WEBHOOK_SECRET: your-secret-here
   GOOGLE_SHEET_ID: your-sheet-id-here
   GOOGLE_SHEET_TAB_NAME: Complaints Register
   ```

4. **Upload Credentials:**
   - In Render dashboard, go to **Environment** → **Secret Files**
   - Upload `credentials.json`
   - Mount path: `/etc/secrets/credentials.json`

5. **Deploy!**
   - Click **Apply**
   - Wait for build (~3-5 minutes)
   - Check logs for any errors

### Option B: Manual Deploy

1. **Create Web Service:**
   - **Name:** `biogas-telegram-bot`
   - **Region:** Oregon (closest to your users)
   - **Branch:** main
   - **Root Directory:** biogas_bot
   - **Runtime:** Python 3

2. **Build Settings:**
   ```
   Build Command:
   pip install -r requirements.txt && python manage.py collectstatic --noinput && python manage.py migrate --noinput

   Start Command:
   gunicorn config.wsgi:application --log-file -
   ```

3. **Add Environment Variables:**
   (Same as Option A, Step 4)

## Step 4: Configure Telegram Webhook

After deployment, set the webhook URL:

```bash
# Replace with your actual values
BOT_TOKEN="123456789:ABCdefGHIjklMNOpqrSTUvwxYZ"
WEBHOOK_URL="https://your-app-name.onrender.com/api/webhook/telegram/"

curl -X POST "https://api.telegram.org/bot${BOT_TOKEN}/setWebhook" \
  -H "Content-Type: application/json" \
  -d "{
    \"url\": \"${WEBHOOK_URL}\",
    \"allowed_updates\": [\"message\"]
  }"
```

**Expected response:**
```json
{
  "ok": true,
  "result": true,
  "description": "Webhook was set successfully"
}
```

### Verify Webhook

```bash
curl "https://api.telegram.org/bot${BOT_TOKEN}/getWebhookInfo"
```

## 🔧 Troubleshooting

### Issue: Webhook not receiving messages

**Check:**
```bash
curl "https://api.telegram.org/bot<TOKEN>/getWebhookInfo"
```

**Look for:**
- `url` matches your Render URL
- `has_custom_certificate`: false
- `pending_update_count`: should decrease over time

**Fix:**
```bash
# Delete and reset webhook
curl -X POST "https://api.telegram.org/bot<TOKEN>/deleteWebhook"
# Then set it again
```

### Issue: Google Sheets not updating

**Check logs for:**
- `Google Sheets service unavailable` → credentials.json not found
- `insufficient permissions` → sheet not shared with service account
- `invalid_grant` → credentials expired

**Fix:**
1. Verify `credentials.json` is uploaded to Render
2. Check `GOOGLE_SHEET_ID` is correct
3. Re-share sheet with service account email

### Issue: Duplicate messages in sheet

**Possible causes:**
- Multiple webhook deliveries (Telegram retries on timeout)
- Manual reprocessing of same batch

**Fix:**
- System has built-in deduplication - check logs for "Duplicate message detected"
- Verify `DEDUPLICATION_WINDOW_MINUTES` is set correctly

### Issue: Messages not parsed correctly

**Check:**
- Log entries with `confidence` score
- Warnings in parse output

**Fix:**
- Add new regex patterns to `services/parser.py`
- For MVP, partial parses are acceptable (raw_message always stored)
- Monitor `ParsedMessage.confidence` in admin dashboard

## 📊 Monitoring & Maintenance

### Daily Checks

1. **Check Render logs** for errors
2. **Verify Google Sheet** has new rows
3. **Check deduplication rate** (should be < 20% of total)

### Weekly Tasks

1. **Review parsing confidence** scores
2. **Clean up old logs** (Render has storage limits)
3. **Backup database** (if using PostgreSQL)

### Monthly Maintenance

1. **Rotate credentials** (Telegram token, service account keys)
2. **Update dependencies** (`pip install --upgrade -r requirements.txt`)
3. **Review and optimize** database indexes
